## 1. 项目名称

**flowEdge**（AI 港股智能投研系统）

---

## 2. 项目简介

flowEdge 是一个面向港股市场的 AI 智能交易决策 Agent。用户输入投资想法与标的代码后，系统自动完成数据采集、闸门诊断、场景分类、五位传奇投资人并行辩论、加权投票、风控审核、交易者方程硬检查、模拟下单，以及时间机器回放验证。

与「单次调用大模型给买卖建议」不同，flowEdge 将 **AI 作为核心推理引擎**，嵌入一条可观测、可回放、可学习的多智能体决策流水线。

---

## 3. 解决的问题或应用场景

### 问题

1. AI 投资建议缺乏可解释性，无法展示推理过程与风控依据
2. LLM 容易在 FOMO、恐慌等极端行情给出追涨杀跌建议
3. 多数 Demo 无法用历史数据验证决策对错
4. 黑客松期间港股休市，需要回放模式演示完整闭环

### 应用场景

- 个人投资者：将模糊投资想法结构化为多维度分析
- 投研团队：快速获得多风格对抗性观点
- 比赛演示：展示「投资想法 → 策略生成 → 执行 / 风控 / 反馈」完整 Agent 闭环
- 策略复盘：回到历史交易日，用当日之前的数据决策并回放验证

## 4. 核心功能说明

| 模块 | 功能 |
|---|---|
| 交易台 | 输入港股代码与投资想法，选择分析模式 |
| 闸门诊断 | 数据充足 → 周期可识别 → 高潮风险 → 风控熔断，任一步拒绝则短路 |
| 场景分类 | 识别突破 / 回调 / 超跌 / 恐慌 / 追高等场景，动态分配大师权重 |
| 五位大师辩论 | 巴菲特 / 索罗斯 / 达利欧 / 林奇 / 利弗莫尔 独立给出信号与陈词 |
| 加权投票 | Multiplicative Weights 按场景权重 + 历史胜率聚合信号 |
| 风控评估 | 波动率仓位、止损止盈、熔断检查、凯利仓位 |
| 交易者方程 | R:R ≥ 1.5 且正期望 EV 才放行 |
| 模拟下单 | BUY / SELL / REJECT，模拟盘执行 |
| 时间机器回放 | 回到历史任意一天决策，回放后续真实 K 线验证对错 |
| 学习闭环 | 大师权重根据回放结果自动更新 |
| 交易卷宗 | 每次分析沉淀为可检索案件记录 |
| 调用溯源 | 记录 LLM 调用哈希，决策路径全程可追溯 |

## 5. AI 使用方式与技术方案

> **AI 在本项目中是核心功能**，贯穿推理、辩论、裁决与学习闭环；规则引擎负责硬约束与兜底。

| 能力 | 是否使用 | 具体作用 |
|---|---|---|
| 大语言模型（LLM） | ✅ 核心 | 豆包驱动 5 位大师独立分析，生成结构化信号与陈词 |
| 多智能体（Multi-Agent） | ✅ 核心 | 5 位大师并行推理 + `court-runner` 编排调度 |
| 提示词工程 | ✅ 核心 | 每位大师独立 persona、投资哲学、JSON Schema |
| 自动化工作流 | ✅ 核心 | 闸门 → 场景路由 → 大师分析 → 风控 → 交易者方程 → 下单，SSE 流式推送 |
| 规则 + AI 混合 | ✅ 辅助 | 闸门、交易者方程、风控为确定性规则；LLM 不可用走 heuristic fallback |
| RAG / 向量检索 | 🔜 规划中 | 判例库语义检索（类型已预留） |
| 模型微调 | ❌ 未使用 | 通过 prompt + 权重学习实现个性化 |

### 技术栈

- 前端：Next.js 16 + React 19 + TailwindCSS v4 + Recharts
- 后端：Next.js Route Handlers（SSE 流式）
- AI：豆包大模型（OpenAI 兼容 API）
- 数据源：腾讯财经港股 K 线 + 实时行情
- 部署：Zeabur

## 6. 使用的数据来源与数据处理方式

| 数据 | 来源 | 说明 |
|---|---|---|
| 港股日 K 线 | 腾讯财经 `gtimg.cn` | 主数据源，支持前复权 |
| 实时行情 | 腾讯财经接口 | 最新价、涨跌幅、成交量 |
| 技术指标 | 本地计算 | RSI / ATR / ADX / MA / EMA |
| 基本面指标 | Mock + 规则推断 | 架构支持后续接入 QVeris / Futu |

处理流程：拉取 → 5 分钟缓存 → API 失败降级 mock → 纯函数计算指标 → 回放模式严格切割历史数据 → 案件与 trace 本地持久化。

## 7. 产品运行方式、访问方式或演示方式

### 本地运行

```bash
git clone https://github.com/jessicaruan6688-byte/flowedge-ai-trading-agent.git
cd flowedge-ai-trading-agent
npm install
npm run dev
# 访问 http://localhost:3000
```

### 环境变量（Zeabur / 本地 `.env.local`）

```env
DOUBAO_API_KEY=your-doubao-api-key
DOUBAO_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
DOUBAO_MODEL=doubao-seed-2-1-pro-260628
AI_TIMEOUT_MS=60000
INITIAL_PAPER_BALANCE=1000000
```

### 推荐演示路径（约 5 分钟）

1. 交易台选择预设案例「0700 腾讯 恐慌抄底」
2. 分析庭观察 SSE 流式点亮 8 个决策节点
3. 查看 5 位大师独立陈词与信号
4. 演示「0981 中芯 FOMO 追高」闸门 REJECT 案例
5. 时间机器回放验证决策后 5 日走势
6. 大师战绩页观察权重变化

## 8. 项目创新点

1. **投资法庭隐喻 + 五位大师对抗**：五种投资哲学实时辩论
2. **两阶段闸门架构**：高潮 / FOMO 行情直接短路，节省 Token
3. **交易者方程硬规则**：R:R ≥ 1.5 且正期望才放行，LLM 无法绕过
4. **时间机器回放**：用历史回放证明 Agent 不是事后诸葛亮
5. **学习闭环权重**：大师错了降权、对了升权
6. **全程 SSE 可视化**：评委实时看到 Agent 思考路径

## 10. 风险提示、局限性或未来改进方向

### 风险提示

- 本项目不构成投资建议，所有输出仅供研究与演示
- AI 可能产生幻觉，陈词内容需人工复核
- 历史回放 ≠ 未来表现

### 局限性

1. 黑客松期间港股休市，采用历史回放模式
2. 数据源主依赖腾讯财经，部分基本面为 mock
3. 判例检索未上线，长期记忆以权重学习为主
4. 仅模拟盘，未对接券商 API

### 未来改进

- 接入 QVeris / Futu OpenAPI
- 实现判例库 RAG
- 支持 A 股 / 美股多市场
- 增加批量回测与鲁棒性评分
