## 2. 项目简介

flowEdge 是一个面向港股市场的 AI 智能交易决策 Agent。它模拟真实「投资委员会」的工作流：用户输入投资想法与标的代码后，系统自动完成数据采集、闸门诊断、场景分类、五位传奇投资人并行辩论、加权投票、风控审核、交易者方程硬检查、模拟下单，以及时间机器回放验证。

与「单次调用大模型给买卖建议」不同，flowEdge 将 **AI 作为核心推理引擎**，嵌入一条可观测、可回放、可学习的多智能体决策流水线。评委可实时看到 Agent 在每一步的思考路径。

## 3. 解决的问题或应用场景

### 问题

1. **AI 投资建议缺乏可解释性**：多数产品只输出结论，无法展示推理过程与风控依据。
2. **缺乏纪律约束**：LLM 容易在 FOMO、恐慌等极端行情给出追涨杀跌建议。
3. **无法验证历史表现**：很多 Demo 是「事后诸葛亮」，没有用历史数据验证决策对错。
4. **港股周末休市**：黑客松期间港股不开盘，需要回放模式演示完整闭环。

### 应用场景

- 个人投资者：将模糊的投资想法结构化为多维度分析
- 投研团队：快速获得多风格（价值 / 宏观 / 成长 / 技术）的对抗性观点
- 比赛演示：展示「投资想法 → 策略生成 → 执行 / 风控 / 反馈」完整 Agent 闭环
- 策略复盘：回到历史任意交易日，用当日之前的数据做决策并回放验证

## 4. 核心功能说明

| 模块 | 功能 |
|---|---|
| **交易台** | 输入港股代码与投资想法，选择分析模式（现货 / 波段 / 事件 / 情绪） |
| **闸门诊断** | 四步顺序闸门：数据充足 → 周期可识别 → 高潮风险 → 风控熔断；任一步拒绝则短路，不浪费 Token |
| **场景分类** | 自动识别突破 / 回调 / 超跌 / 区间 / 恐慌 / 追高等 10 种场景，动态分配大师权重 |
| **五位大师辩论** | 巴菲特 / 索罗斯 / 达利欧 / 林奇 / 利弗莫尔 基于各自投资哲学独立给出信号与陈词 |
| **加权投票** | Multiplicative Weights 算法，按场景权重 + 历史胜率聚合信号 |
| **风控评估** | 波动率仓位调整、止损止盈、熔断检查、凯利仓位 |
| **交易者方程** | 盈亏比 R:R ≥ 1.5 且正期望 EV 才放行，硬规则不可被 LLM 绕过 |
| **模拟下单** | BUY / SELL / REJECT 裁决，模拟盘执行并更新组合 |
| **时间机器回放** | 回到历史任意一天，只用那天之前的数据决策，回放后续真实 K 线验证对错 |
| **学习闭环** | 大师权重根据回放结果自动更新，胜率高的风格在同类场景话语权更大 |
| **交易卷宗** | 每次分析沉淀为可检索的案件记录 |
| **调用溯源** | 记录每次 LLM 调用的输入输出哈希，决策路径全程可追溯 |

### 决策路径示意

```
数据充足? → 周期可识别? → 高潮风险? → 风控熔断? → 场景分类 → 5大师辩论 → 交易者方程 → BUY/SELL/REJECT
   ✓           ✓           ✗           ✗       回调买入   3多1空1中性   R:R=2.3✓      BUY
```

## 5. AI 使用方式与技术方案

> **AI 在本项目中是核心功能**，贯穿推理、辩论、裁决与学习闭环；规则引擎负责硬约束与兜底。

### 5.1 AI 能力使用清单

| 能力 | 是否使用 | 具体作用 |
|---|---|---|
| **大语言模型（LLM）** | ✅ 核心 | 豆包 `doubao-seed-2-1-pro` 驱动 5 位大师独立分析，生成结构化信号与陈词 |
| **多智能体（Multi-Agent）** | ✅ 核心 | 5 位大师并行推理 + 编排器 `court-runner` 统一调度 |
| **提示词工程（Prompt Engineering）** | ✅ 核心 | 每位大师有独立 persona、投资哲学、输出 JSON Schema |
| **工具调用 / 自动化工作流** | ✅ 核心 | 闸门 → 场景路由 → 大师分析 → 风控 → 交易者方程 → 下单，SSE 流式推送 |
| **规则 + AI 混合** | ✅ 辅助 | 闸门诊断、交易者方程、风控熔断为确定性规则；LLM 挂了走 heuristic fallback |
| **RAG / 向量检索** | 🔜 规划中 | 判例库语义检索（类型已预留，当前 Demo 以场景权重学习为主） |
| **模型微调** | ❌ 未使用 | 通过 prompt + 权重学习实现个性化，无需微调 |
| **多模态 / 语音** | ❌ 未使用 | 当前聚焦文本 + 行情数据 |
| **智能体记忆** | ✅ 部分实现 | 大师权重按场景动态更新，形成「谁在这个场景更靠谱」的长期记忆 |

### 5.2 技术架构

```
用户输入（交易台）
    ↓
court-runner 编排器（SSE 流式）
    ├─ data-source      腾讯财经港股 K 线 + 技术指标
    ├─ gate-engine      四步闸门（纯函数，无 LLM）
    ├─ scenario-router  场景分类 + 权重分配
    ├─ agents/          5 位大师（各调一次 LLM，并行）
    ├─ risk-engine      风控评估
    ├─ trader-equation  交易者方程硬检查
    ├─ portfolio-manager 模拟盘执行
    ├─ replay-engine    时间机器回放
    └─ weight-store     学习闭环权重更新
```

### 5.3 技术栈

- **前端**：Next.js 16 (App Router) + React 19 + TailwindCSS v4 + Recharts
- **后端**：Next.js Route Handlers，SSE 流式输出
- **AI**：豆包大模型（OpenAI 兼容 API），支持替换为任意兼容端点
- **部署**：Zeabur（Next.js 自动识别）

### 5.4 LLM 调用细节

- 每位大师一次独立 LLM 调用，返回结构化 JSON（信号 / 置信度 / 陈词 / 理由）
- `ai-service.ts` 封装 OpenAI 兼容协议，含三级 JSON 解析容错
- LLM 不可用时自动降级为规则版 heuristic，模式标记为 `fallback`，不伪造 AI 结果
- 所有调用记录 trace（输入输出哈希脱敏），可在「调用溯源」页查看

## 6. 使用的数据来源与数据处理方式

### 6.1 数据源

| 数据 | 来源 | 说明 |
|---|---|---|
| 港股日 K 线 | 腾讯财经 `gtimg.cn` | 主数据源，大陆可访问；支持前复权 |
| 实时行情 | 腾讯财经接口 | 最新价、涨跌幅、成交量 |
| 技术指标 | 本地计算 | RSI / ATR / ADX / MA / EMA，基于 K 线实时计算 |
| 基本面指标 | Mock + 规则推断 | Demo 阶段部分指标由规则生成，架构支持后续接入 QVeris / Futu |

### 6.2 数据处理流程

1. **拉取**：按股票代码规范化（如 `0700.HK` → `hk00700`），请求腾讯财经 API
2. **缓存**：K 线与行情 5 分钟内存缓存，减少重复请求
3. **降级**：API 失败时自动切换 mock 数据，保证 Demo 可演示
4. **指标计算**：纯函数计算技术因子，供闸门与场景分类使用
5. **时间切割**：回放模式下严格只用决策日之前的 K 线，防止未来数据泄露
6. **持久化**：案件、组合状态、trace 记录本地 JSON 存储（`.flowedge/`）

## 7. 产品运行方式、访问方式或演示方式

### 7.1 本地运行

```bash
# 1. 克隆仓库
git clone https://github.com/jessicaruan6688-byte/flowedge-ai-trading-agent.git
cd flowedge-ai-trading-agent

# 2. 安装依赖
npm install

# 3. 配置环境变量
cp .env.example .env.local
# 编辑 .env.local 填入 DOUBAO_API_KEY（不配置也可跑 mock 演示模式）

# 4. 启动开发服务器
npm run dev

# 5. 浏览器访问 http://localhost:3000
```

### 7.2 环境变量

```env
DOUBAO_API_KEY=your-doubao-api-key
DOUBAO_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
DOUBAO_MODEL=doubao-seed-2-1-pro-260628
AI_TIMEOUT_MS=60000
INITIAL_PAPER_BALANCE=1000000
```

## 8. 项目创新点

1. **投资法庭隐喻 + 五位大师对抗**：不是单一 LLM 给答案，而是五种投资哲学实时辩论，视觉与叙事冲击力强。
2. **两阶段闸门架构**：先诊断「该不该分析」，再进入大师辩论，高潮/FOMO 行情直接短路，节省 Token 且体现纪律。
3. **交易者方程硬规则**：R:R ≥ 1.5 且正期望才放行，LLM 无法绕过数学约束。
4. **时间机器回放**：黑客松期间港股休市，用历史回放证明 Agent 不是事后诸葛亮。
5. **学习闭环权重**：大师错了降权、对了升权，形成场景化的「谁更靠谱」记忆，接近对冲基金 PM 考核机制。
6. **全程 SSE 可视化**：评委实时看到 Agent 思考路径，不是黑盒输出。

## 10. 风险提示、局限性或未来改进方向

### 风险提示

- **本项目不构成投资建议**，所有输出仅供研究与演示，模拟盘结果不代表真实收益。
- **AI 可能产生幻觉**，尽管有规则兜底与 fallback 机制，陈词内容仍需人工复核。
- **历史回放 ≠ 未来表现**，时间机器验证的是过去决策逻辑，不保证未来有效。

### 当前局限性

1. **港股周末休市**：比赛期间无法 live 行情，采用历史回放模式演示。
2. **数据源单一**：主依赖腾讯财经，部分基本面为 mock，未接入 QVeris 付费能力。
3. **判例检索未上线**：长期记忆目前以权重学习为主，语义判例库待实现。
4. **单市场**：当前聚焦港股，未覆盖 A 股 / 美股。
5. **无实盘执行**：仅模拟盘，未对接券商 API。

### 未来改进方向

- 接入 QVeris / Futu OpenAPI，丰富港股基本面与新闻数据
- 实现判例库 RAG：Bear 引用历史相似案件，Judge 做适用性分析
- 支持 A 股 / 美股多市场 adapter
- 增加回测批量模式与鲁棒性评分
- 对接 Paper Trading API 或券商 sandbox
